# Решение квадратного уравнения демпфированным методом Ньютона

Небольшой проект на C++, который находит вещественные корни уравнения

```math
a x^2 + b x + c = 0
```

с использованием:

- анализа экстремума параболы
- демпфированного метода Ньютона
- стабилизации через арктангенс

## Что умеет программа

- решает квадратные уравнения
- корректно обрабатывает линейный случай
- определяет количество вещественных корней по экстремуму
- уточняет корни численным методом
- снижает риск неустойчивых шагов Ньютона за счёт демпфирования

## Математическая модель

Рассматривается функция

```math
f(x) = a x^2 + b x + c
```

Для квадратной функции экстремум находится в точке

```math
x^\* = -\frac{b}{2a}
```

а значение функции в экстремуме равно

```math
f(x^\*)
```

Это позволяет определить число вещественных корней без прямого использования дискриминанта:

- если $f(x^\*) = 0$, то существует один двукратный корень
- если $a > 0$ и $f(x^\*) > 0$, вещественных корней нет
- если $a > 0$ и $f(x^\*) < 0$, есть два вещественных корня
- для $a < 0$ логика симметрична

После этого корни уточняются демпфированным методом Ньютона.

## Преобразование через арктангенс

Вместо прямого применения метода Ньютона к $f(x)$ программа использует преобразованную функцию

```math
\varphi(x) = \arctan \left(s \, f(x)\right)
```

где $s$ — масштабный коэффициент.

Это нужно для того, чтобы:

- сжать слишком большие значения функции
- сделать шаги устойчивее вдали от корня
- уменьшить вероятность неудачного перелёта

Производная имеет вид

```math
\varphi'(x) = \frac{s f'(x)}{1 + \left(s f(x)\right)^2}
```

где

```math
f'(x) = 2 a x + b
```

## Демпфированный шаг Ньютона

Сначала считается стандартный шаг Ньютона:

```math
\delta = \frac{\varphi(x)}{\varphi'(x)}
```

Затем применяется коэффициент демпфирования:

```math
\lambda = \frac{\arctan\left(|\delta|\right)}{|\delta|}
```

и новая итерация вычисляется по формуле

```math
x_{k+1} = x_k - \lambda \delta
```

Смысл этого шага:

- маленький шаг -> метод почти совпадает с классическим Ньютоном
- большой шаг -> шаг автоматически уменьшается

## Как ищутся корни

Если у уравнения два вещественных корня, точка экстремума делит параболу на две монотонные ветви:

- левую
- правую

Программа:

1. берёт точку экстремума
2. расширяет область влево, пока не найдёт смену знака функции
3. расширяет область вправо, пока не найдёт смену знака функции
4. использует середину найденного интервала как начальное приближение

После этого запускается демпфированный метод Ньютона для каждой ветви.

## Особые случаи

- если $a = 0$, решается линейное уравнение $b x + c = 0$
- если $a = 0$, $b = 0$, $c = 0$, решений бесконечно много
- если $a = 0$, $b = 0$, $c \neq 0$, решений нет
- программа ищет только вещественные корни

## Проверенные случаи

- $1x^2 - 3x + 2 = 0$ -> два корня: $1$ и $2$
- $1x^2 + 2x + 1 = 0$ -> один двукратный корень: $-1$
- $1x^2 + 1 = 0$ -> вещественных корней нет
- $2x - 8 = 0$ -> линейный случай, корень $4$

## Что реализовано в коде

- `polynomial(...)` вычисляет значение квадратного трёхчлена
- `derivative(...)` вычисляет производную
- `transformed_function(...)` строит $\arctan(s f(x))$
- `transformed_derivative(...)` вычисляет производную преобразованной функции
- `damping_from_step(...)` считает коэффициент демпфирования
- `damped_newton(...)` выполняет итерационный процесс
