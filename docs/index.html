ядро метода

устойчивость важнее бравады
$$a x^2 + b x + c = 0$$

Экстремум

$$x^\* = -\frac{b}{2a}$$

Сначала выясняем структуру корней через геометрию параболы.

Сглаживание

$$\varphi(x)=\arctan(s f(x))$$

Арктангенс сжимает большие значения и делает шаги осторожнее.

Скорость на хорошем старте обычный Ньютон
Устойчивость на плохом старте демпфированный Ньютон

идея

Сначала понимаем форму функции, потом уже запускаем итерации

Ключевая мысль проекта проста: не бросать Ньютон вслепую в уравнение, а сначала определить характер задачи через экстремум квадратичной функции.

Формула $$f(x)=a x^2+b x+c$$

шаг метода

Итерация в компактном виде

delta = phi / phi'
lambda = atan(|delta|) / |delta|
x = x - lambda * delta

На хороших стартах это почти классический Ньютон. На плохих — уже более осторожная схема.

смысл arctan

Не ускорять любой ценой, а удерживать итерации в руках

Здесь демпфирование нужно не для красоты. Оно снижает шанс грубого перелёта, когда чистый Ньютон на сложной функции делает слишком агрессивный шаг.

алгоритм

Четыре шага, без лишнего тумана

01

Проверка типа задачи

Если \(a = 0\), решаем линейное уравнение. Иначе остаёмся в квадратичном случае.

02

Поиск экстремума

Вычисляем \(x^\* = -b / 2a\) и по знаку \(f(x^\*)\) понимаем, сколько существует корней.

03

Стартовые приближения

Для двух корней строим интервалы со сменой знака слева и справа от экстремума и берём их середины.

04

Численное уточнение

Дальше работает либо обычный шаг Ньютона, либо демпфированный вариант с более мягкой динамикой.

графики

Как падает невязка по итерациям

На графиках показана \(\log_{10}(|f(x_n)|)\). Чем быстрее кривая идёт вниз, тем быстрее метод приближается к корню.
